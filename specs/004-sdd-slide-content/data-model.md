# Data Model: SDDスライドコンテンツ構造

## スライド構成一覧

本ドキュメントはスライドの論理構造を定義する。各スライドのタイトル・コンテンツタイプ・主要メッセージを記載し、実装時のガイドとする。

### 全体構造

```
H1: Spec Kitではじめる仕様駆動開発入門
├── H2: 仕様駆動開発(SDD)とは（5スライド）
│   ├── H3: バイブコーディングの課題
│   ├── H3: SDDとは
│   ├── H3: 従来の開発 vs SDD
│   ├── H3: レシピと料理で例えると
│   └── H3: SDDの基本フロー
├── H2: SDDのツール（1スライド）
│   └── H3: SDDのツール紹介
├── H2: Spec Kit入門（10スライド）
│   ├── H3: GitHub Spec Kit紹介
│   ├── H3: Spec Kitのワークフロー
│   ├── H3: Constitution
│   ├── H3: Specify
│   ├── H3: Clarify
│   ├── H3: Plan
│   ├── H3: Tasks
│   ├── H3: Analyze
│   ├── H3: Implement
│   └── H3: Non-Goalsの重要性
├── H2: 実践例（3スライド）
│   ├── H3: hachimokuプロジェクト
│   ├── H3: 仕様書の例
│   └── H3: 仕様間の依存関係
└── H2: まとめ（3スライド）
    ├── H3: SDDの欠点と注意点
    ├── H3: まとめ
    └── H3: 参考リンク
```

**スライド総数**: 22枚（タイトル含まず）+ セクション区切り5枚 = 27枚

### 時間配分

| セクション | スライド数 | 目安時間 | 備考 |
|---|---|---|---|
| 仕様駆動開発(SDD)とは | 5 | 5〜6分 | 課題提起→定義→対比→たとえ話→フロー |
| SDDのツール | 1 | 1〜2分 | さらっと紹介 |
| Spec Kit入門 | 10 | 10〜12分 | 各ステップ1分弱 |
| 実践例 | 3 | 3〜4分 | literalinclude はさらっと流す |
| まとめ | 3 | 2〜3分 | 欠点→要約→リンク |
| **合計** | **22** | **21〜27分** | 目標25分 |

### セクション1: 仕様駆動開発(SDD)とは

#### S-02: バイブコーディングの課題

- **見出し**: H3 `バイブコーディングの課題`
- **コンテンツ**: 箇条書き（4〜5項目）
- **主要メッセージ**: AIにコードを任せると品質にばらつきが出る
- **タイプ**: 箇条書き
- **出典**: 不要（一般的な課題認識）

#### S-03: SDDとは

- **見出し**: H3 `仕様駆動開発(SDD)とは`
- **コンテンツ**: 箇条書き（3〜4項目）+ 出典リンク
- **主要メッセージ**: コードより先に仕様書を書く開発手法。仕様書が真実の源泉
- **タイプ**: 箇条書き
- **出典**: GitHub spec-kit `spec-driven.md`、Wikipedia SDD記事

#### S-04: 従来の開発 vs SDD

- **見出し**: H3 `従来の開発 vs SDD`
- **コンテンツ**: list-table（2列）
- **主要メッセージ**: SDDは仕様書とコードの関係を逆転させる
- **タイプ**: テーブル
- **Clarification反映**: 「仕様を書く過程で曖昧な考えが明確化」を追加
- **出典**: 不要（対比表は構成的）

#### S-05: レシピと料理で例えると

- **見出し**: H3 `レシピと料理で例えると`
- **コンテンツ**: list-table（3列）
- **主要メッセージ**: 曖昧なレシピ→ばらつき、詳細なレシピ→再現性
- **タイプ**: テーブル
- **出典**: 不要（アナロジー）

#### S-06: SDDの基本フロー

- **見出し**: H3 `SDDの基本フロー`
- **コンテンツ**: mermaid flowchart（TD方向、5ステップ）
- **主要メッセージ**: Specify→Plan→Tasks→Implement→Review の流れ

## コンテンツの状態遷移

```
既存 index.md → outline.txt で構成を確認 → 仕様に基づき再構成
  ├── 残すスライド: レシピの例え、基本フロー、欠点・注意点
  ├── 移動するスライド: バイブコーディング(→SDD配下)、Spec Kit紹介(→入門配下)
  ├── 削除するスライド: AIとの組み合わせの利点、類似手法との比較
  ├── 統合するスライド: 適している/いない場合 → 欠点と注意点
  └── 新規追加: SDD定義、各ワークフローステップ、hachimoku実践例
```

## バリデーションルール

| ルール | 対象 | 基準 |
|---|---|---|
| スライドタイトル | 全H3 | 15文字以内 |
| 箇条書き文字数 | 全箇条書き項目 | 25文字以内（全角換算） |
| 箇条書き項目数 | 全スライド | 6項目以内 |
| テーブル列数 | 全list-table | 3列以内 |
| コード行数 | 全コードブロック | 10行以下 |
| ダイアグラム | フロー図 | mermaid ディレクティブ使用 |
| テーブル記法 | 全テーブル | list-table ディレクティブ使用 |
| 出典 | 用語・概念の解説 | ハイパーリンク必須 |
